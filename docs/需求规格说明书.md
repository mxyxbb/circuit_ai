# CircuitAI 电路仿真器 - 需求规格说明书

**版本**: 1.0
**日期**: 2026-04-11

---

## 1. 产品概述

CircuitAI 是一款面向电力电子与电路分析领域的桌面级电路仿真软件。基于改进节点分析法 (MNA) 求解电路方程，采用定步长后向欧拉法处理动态元件，通过分段线性 (PWL) 模型处理非线性器件。前端使用 ImGui/ImPlot 提供高性能即时渲染界面，整体架构遵循 MVVM 模式实现前后端解耦。

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| 前后端物理隔离 | 后端严禁引入任何 UI 库；前端严禁直接操作 MNA 矩阵 |
| 数据驱动 | 通过 netlist 文件定义电路拓扑，运行时解析 |
| 多线程异步 | 仿真引擎运行在独立线程，UI 保持流畅响应 |
| 零拷贝渲染 | 波形数据通过环形缓冲区直传 ImPlot，避免帧间内存分配 |

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| MNA | Modified Nodal Analysis，改进节点分析法 |
| PWL | Piece-Wise Linear，分段线性模型 |
| SPSC | Single Producer Single Consumer，单生产者单消费者无锁队列 |
| Probe | 探针，指定需要观测的节点电压或支路电流信号 |

## 2. 功能需求

### 2.1 仿真引擎 (Simulation Engine)

#### 2.1.1 求解器核心

| 需求项 | 规格说明 |
|--------|----------|
| 求解方法 | 改进节点分析法 (MNA)，构建 [A][x]=[b] 线性系统 |
| 线性代数 | 基于 Eigen 库的 LU 分解求解稠密矩阵 |
| 离散化 | 定步长后向欧拉法 (Backward Euler)，用于电容/电感伴随模型 |
| 非线性收敛 | 分段线性迭代法，最大迭代 20 次，收敛阈值 |Δx| < 1e-9 |
| 状态突变保护 | 开关/二极管状态翻转时触发重新求解，防止逻辑震荡 |
| 矩阵规模 | 初期支持 ≤100 节点的稠密矩阵，预留稀疏矩阵升级接口 |

#### 2.1.2 仿真调度

| 需求项 | 规格说明 |
|--------|----------|
| 时间推进 | 定步长，Δt 由用户设定（默认 1μs） |
| 仿真终止 | 达到用户指定的结束时间 t_end 后自动停止 |
| 运行控制 | 支持启动 (Run)、暂停 (Pause)、重置 (Reset) 三种状态 |
| 探针采样 | 引擎按固定采样率（每 N 步采一点）向 UI 推送数据 |

#### 2.1.3 多线程模型

```
┌─────────────────┐         ┌──────────────────┐
│  Simulation     │  SPSC   │  UI Thread       │
│  Thread         │ RingBuf │  (Main Loop)     │
│                 │───────> │                  │
│  while(running) │         │  ViewModel::     │
│    step()       │         │    update()      │
│    if(sample)   │         │  ScopeView::     │
│      push(data) │         │    render()      │
└─────────────────┘         └──────────────────┘
```

- 引擎线程：独立 `std::thread`，循环执行 `simulator::step()`
- 数据桥接：基于 `std::atomic` 的 SPSC 无锁环形缓冲区
- 停止控制：通过 `std::atomic<bool>` 信号量实现优雅停止

### 2.2 器件模型 (Component Models)

所有器件继承自 `BaseComponent` 基类，实现统一的 `stamp()` 接口。

#### 2.2.1 无源器件 (Passives)

##### 电阻 (Resistor)

| 属性 | 说明 |
|------|------|
| 参数 | R（阻值，单位 Ω） |
| 端口 | N+, N- |
| MNA 行为 | 电导 G=1/R 盖章：G[i][i]+=G, G[j][j]+=G, G[i][j]-=G, G[j][i]-=G |
| 非线性 | 无 |
| 储能 | 无 |

##### 电容 (Capacitor)

| 属性 | 说明 |
|------|------|
| 参数 | C（容值，单位 F） |
| 端口 | N+, N- |
| MNA 行为 | 后向欧拉伴随模型：G_eq=C/Δt 电阻盖章 + 历史电流源 I_hist=(C/Δt)·V_prev |
| 非线性 | 无 |
| 储能 | 是，每步收敛后记录 V_prev |

##### 电感 (Inductor)

| 属性 | 说明 |
|------|------|
| 参数 | L（感值，单位 H） |
| 端口 | N+, N- |
| MNA 行为 | 后向欧拉 Norton 伴随模型：等效电导 G_eq=Δt/L 盖章 + 历史电流源 I_hist=I_prev。与电容伴随模型对称，无需额外 MNA 变量 |
| 非线性 | 无 |
| 储能 | 是，每步收敛后记录 I_prev（电感电流） |

##### 理想变压器 (Transformer)

| 属性 | 说明 |
|------|------|
| 参数 | n（匝数比，V1/V2=n） |
| 端口 | N1+, N1-（初级）, N2+, N2-（次级） |
| MNA 行为 | 基于受控源模型，需要 1 个额外电流变量 I1，约束 V1=n·V2，电流关系 I2=-n·I1 由盖章系数自动满足 |
| 非线性 | 无 |
| 储能 | 无 |

#### 2.2.2 半导体器件 (Semiconductors)

##### 理想二极管 (Ideal Diode)

| 属性 | 说明 |
|------|------|
| 参数 | 无（内建 R_on=1mΩ, R_off=1GΩ） |
| 端口 | Anode (阳极), Cathode (阴极) |
| MNA 行为 | PWL 模型。导通态等效 R_on 盖章，截止态等效 R_off 盖章 |
| 状态机 | ON→OFF：V_ak < -V_threshold; OFF→ON：I_ak > I_threshold |
| 非线性 | 是，需要迭代收敛判定 |

##### 理想开关管 (Ideal Switch)

| 属性 | 说明 |
|------|------|
| 参数 | 无（内建 R_on=1mΩ, R_off=1GΩ） |
| 端口 | Drain (漏极), Source (源极), Gate (栅极), Gate_Ref (栅极参考) |
| MNA 行为 | PWL 模型。Gate 电压高于阈值时导通 (R_on)，否则截止 (R_off) |
| 状态机 | 由外部控制信号驱动，无需迭代 |
| 非线性 | 伪非线性（状态由外部决定，无需收敛判定） |

#### 2.2.3 激励源 (Sources)

##### 直流电压源 (DC Voltage Source)

| 属性 | 说明 |
|------|------|
| 参数 | V（电压值，单位 V） |
| 端口 | N+, N- |
| MNA 行为 | 申请额外电流变量 i_v，盖章 B[i][i_v]=1, C[i_v][i]=1, b[i_v]=V |
| 非线性 | 无 |

##### 直流电流源 (DC Current Source)

| 属性 | 说明 |
|------|------|
| 参数 | I（电流值，单位 A） |
| 端口 | N+, N- |
| MNA 行为 | 直接注入电流：b[N+]+=I, b[N-]-=I |
| 非线性 | 无 |

##### 方波电压源 (Square Wave Source)

| 属性 | 说明 |
|------|------|
| 参数 | freq（频率 Hz）, duty（占空比 0~1）, Vhigh（高电平 V）, Vlow（低电平 V） |
| 端口 | N+, N- |
| MNA 行为 | 继承 DC Voltage Source 的盖章方式，每步根据 t 计算 V(t) |
| 波形公式 | V(t) = (mod(t·freq, 1.0) < duty) ? Vhigh : Vlow |
| 非线性 | 无 |

##### 阶跃波电压源 (Step Source)

| 属性 | 说明 |
|------|------|
| 参数 | V0（初始电压 V）, V1（阶跃电压 V）, tdelay（延迟时间 s） |
| 端口 | N+, N- |
| MNA 行为 | 继承 DC Voltage Source 的盖章方式，每步根据 t 计算 V(t) |
| 波形公式 | V(t) = (t < tdelay) ? V0 : V1 |
| 非线性 | 无 |

##### 正弦交流电压源 (SIN Source)

| 属性 | 说明 |
|------|------|
| 参数 | VOFF（直流偏置 V）, VAMPL（峰值振幅 V）, FREQ（频率 Hz）, PHASE（初始相位 rad） |
| 端口 | N+, N- |
| MNA 行为 | 继承 DC Voltage Source 的盖章方式，每步根据 t 计算 V(t) |
| 波形公式 | V(t) = Voff + Vampl × sin(2π × freq × t + phase) |
| 非线性 | 无 |

### 2.3 网表解析器 (Netlist Parser)

#### 2.3.1 网表格式规范

采用类 SPICE 的文本格式，易于手工编写和程序生成。

```
* Comment line (starts with *)
* Component definitions:
*   R<name>  N+  N-  <value>
*   C<name>  N+  N-  <value>
*   L<name>  N+  N-  <value>
*   V<name>  N+  N-  DC <value>
*   I<name>  N+  N-  DC <value>
*   V<name>  N+  N-  SQUARE freq=<f> duty=<d> Vhigh=<vh> Vlow=<vl>
*   V<name>  N+  N-  STEP V0=<v0> V1=<v1> tdelay=<td>
*   V<name>  N+  N-  SIN VOFF=<vo> VAMPL=<va> FREQ=<f> PHASE=<p>
*   D<name>  Anode  Cathode
*   S<name>  Drain  Source  Gate  GateRef
*   TX<name> N1+  N1-  N2+  N2-  ratio=<n>
*
* Control commands:
*   .TRAN <dt> <t_end>            ; Transient analysis
*   .PROBE V(<node>) I(<comp>)    ; Signals to observe
*   .END

* Example: RC Low-Pass Filter
R1  1  2  1k
C1  2  0  1u
V1  1  0  SQUARE freq=1k duty=0.5 Vhigh=5 Vlow=0
.TRAN 1e-6 0.01
.PROBE V(1) V(2) I(R1)
.END
```

#### 2.3.2 数值后缀支持

| 后缀 | 倍率 | 示例 |
|------|------|------|
| f | 1e-15 | — |
| p | 1e-12 | 10pF → 1e-11 |
| n | 1e-9 | 100nH → 1e-7 |
| u / μ | 1e-6 | 1uF → 1e-6 |
| m | 1e-3 | 10mH → 1e-2 |
| k | 1e3 | 1kΩ → 1e3 |
| Meg | 1e6 | 1MegΩ → 1e6 |

#### 2.3.3 解析器需求

| 需求项 | 规格说明 |
|--------|----------|
| 错误报告 | 报告行号与具体错误原因 |
| 节点编号 | 支持整数编号，节点 0 固定为 GND（参考地） |
| 组件命名 | 首字母决定器件类型（R/C/L/V/I/D/S/TX），后跟标识符 |
| 大小写 | 器件类型前缀不区分大小写 |

### 2.4 图形界面 (UI)

整体布局基于 ImGui DockSpace 停靠空间，支持多窗口自由拖拽。

#### 2.4.1 波形显示器 (ScopeView) — 一期核心

波形显示功能由 Scope 模块提供（详见 2.5 节），以多坐标区示波器形态呈现，是一期重点实现的面板。

#### 2.4.2 仿真设置器 (SettingsView) — 保留框架

| 需求项 | 规格说明 |
|--------|----------|
| 控制按钮 | Run / Pause / Reset |
| 参数设置 | 步长 Δt、终止时间 t_end |
| 状态显示 | 当前仿真时间 t、仿真状态 (Running/Paused/Stopped) |
| 文件加载 | Netlist 文件路径选择（.TRAN 和 .PROBE 从中解析） |

#### 2.4.3 器件选择器 (PaletteView) — Phase 3

| 需求项 | 规格说明 |
|--------|----------|
| 器件列表 | 按类别列出所有支持的器件类型（无源、半导体、激励源、特殊） |
| 拖拽放置 | 从列表拖拽到 SchematicView 画布，松开鼠标即在落点创建实例 |
| 搜索过滤 | 输入框实时过滤器件名称 |
| 快捷提示 | 悬停显示器件说明（引脚数、参数说明） |

#### 2.4.4 电路画布 (SchematicView) — Phase 3

| 需求项 | 规格说明 |
|--------|----------|
| 无限画布 | 支持平移（中键拖拽或 Alt+左键）和缩放（滚轮） |
| 网格对齐 | 20px 网格，器件位置吸附到网格 |
| 放置器件 | 从 PaletteView 拖入；右键菜单也可选择放置 |
| 参数编辑 | 双击器件打开属性对话框，编辑所有参数 |
| 选择与移动 | 左键单击选中，拖拽已选器件移动；点击空白处取消选中 |
| 连线绘制 | 悬停引脚时高亮；左键点击引脚开始连线，再点击目标引脚完成 |
| 删除 | 选中后按 Delete 删除器件（关联连线一并删除） |
| GND 符号 | 特殊单引脚器件，所有连接到 GND 符号的网络自动分配节点 0 |
| 构建仿真 | "Build & Run" 按钮自动生成网表 → 加载到仿真器并启动仿真 |
| 生成网表 | 拓扑 → MNA 节点编号（Union-Find 连通分量算法）→ SPICE 格式网表字符串 |

### 2.5 示波器 (Scope) — 一期核心

Scope 是独立于上述面板的波形显示模块，提供可配置的多坐标区波形观测能力。

#### 2.5.1 功能概述

Scope 模拟真实示波器的多通道显示体验。用户可以创建多个独立的坐标区 (Plot)，每个坐标区可以绑定多个探针信号（MUX），实现信号的灵活组合与分组显示。

```
┌─────────────────── Scope ──────────────────────┐
│  ┌───────────── Plot 0 ──────────────────────┐  │
│  │  MUX: V(1) ──── V(2)                      │  │
│  │  ═════════════════════════════════════     │  │
│  └───────────────────────────────────────────┘  │
│  ┌───────────── Plot 1 ──────────────────────┐  │
│  │  MUX: I(R1)                               │  │
│  │  ═════════════════════════════════════     │  │
│  └───────────────────────────────────────────┘  │
│  [+ Insert Plot Above] [+ Insert Plot Below]    │
└─────────────────────────────────────────────────┘
```

#### 2.5.2 坐标区管理 (Plot)

| 操作 | 触发方式 | 说明 |
|------|----------|------|
| 初始状态 | 程序启动 | Scope 默认包含 1 个空坐标区 |
| Insert Plot Above | 右键菜单 / 按钮 | 在当前坐标区上方插入新的空坐标区 |
| Insert Plot Below | 右键菜单 / 按钮 | 在当前坐标区下方插入新的空坐标区 |
| Delete Plot | 右键菜单 | 删除当前坐标区（Scope 至少保留 1 个坐标区） |
| 选中高亮 | 鼠标点击坐标区 | 当前选中的坐标区边框高亮，Insert/Delete 操作基于选中坐标区 |

#### 2.5.3 信号 MUX (Multiplexer)

MUX 是绑定在单个坐标区上的信号列表，同一 MUX 中的所有信号共享 X 轴（时间）和 Y 轴。

| 操作 | 说明 |
|------|------|
| 添加信号 | 从 `.PROBE` 定义的可用信号列表中选择，添加到指定坐标区的 MUX |
| 移除信号 | 点击图例中的信号名称或通过右键菜单移除 |
| 信号颜色 | 每个信号分配独立颜色，通过图例区分 |
| 信号显隐 | 点击图例名称切换信号在 Plot 中的可见性 |
| 同一信号复用 | 同一个 Probe 信号可以被添加到多个坐标区中 |

#### 2.5.4 轴行为

| 属性 | 说明 |
|------|------|
| X 轴联动 | 所有坐标区的 X 轴（时间轴）同步缩放和平移，用户缩放任意一个 Plot 的时间范围时，其余 Plot 同步跟随 |
| Y 轴独立 | 每个坐标区的 Y 轴独立自动缩放，适应该坐标区 MUX 中信号的幅值范围 |
| Auto-Fit | 支持单坐标区 Auto-Fit 和全局 Auto-Fit |
| 时间光标 | 鼠标悬停时，在所有坐标区同步显示竖直时间光标和精确数值 |
| 滚动模式 | 仿真运行时，所有坐标区自动跟随最新时间点滚动 |

#### 2.5.5 交互需求

| 交互 | 说明 |
|------|------|
| 右键菜单（坐标区空白处） | Insert Plot Above / Insert Plot Below / Delete Plot / Add Signal → 子菜单列出可用信号 |
| 右键菜单（信号图例项） | Remove from Plot / Change Color |
| 拖拽信号到坐标区 | 从 SettingsView 的信号列表拖拽到目标 Plot，自动添加到 MUX |
| 鼠标滚轮 | 缩放当前坐标区的 X/Y 轴 |
| 鼠标中键拖拽 | 平移坐标区视图 |

## 3. 非功能需求

| 类别 | 需求 |
|------|------|
| 平台 | Windows 10+（初期），架构预留 Linux/macOS 移植能力 |

## 4. 开发阶段

### Phase 1 — 管道打通 + 波形显示（当前目标）

1. 搭建 CMake 构建系统，运行空 ImGui 窗口
2. 实现 `MNASolver` 基本矩阵求解
3. 实现 `Resistor`、`DC Voltage Source`、`Current Source`
4. 实现 `Capacitor`（伴随模型验证）
5. 实现 Netlist Parser（基础子集）
6. 实现 `Simulator` 单步执行循环
7. 实现 `SPSCRingBuffer` 和 `ScrollingBuffer`
8. 实现 `MainViewModel` 多线程调度
9. 实现 `ScopeView` 多坐标区 ImPlot 渲染（默认单 Plot，支持 Insert/Delete）
10. 端到端验证：RC 低通滤波器 netlist → 正弦/方波激励 → 充电曲线显示

### Phase 2 — 完整器件 + 交互控制

- 填充所有器件（L, Diode, Switch, Transformer, SquareWave, Step）
- 实现 `SettingsView` 的完整交互
- 仿真状态管理完善

### Phase 3 — 可视化编辑

- 实现 `PaletteView` 器件拖拽
- 实现 `SchematicView` 电路画布
- 可视化拓扑 → Netlist 自动生成
